## Four needs → stack mapping

| Need | What it means for TaskShift | Stack component |
|------|----------------------------|-----------------|
| **Remember across time** | Store O*NET task bundles and each user's personal time weights + scores so they persist. | **Supabase** — `onet_tasks` (reference) + `user_submissions` (personal estimate) |
| **React to an event** | When the user submits a role or clicks "Calculate," fetch the bundle, run the formula, call the LLM, return results. | **n8n** — webhook triggers the full orchestration chain on each submission event |
| **Generate from a mess** | A person types a free-text job title. The system must map it to a structured occupation and write a personalised AI-native role description + roadmap. | **Groq** — two bounded calls. Numbers never generated by LLM. |
| **Give a person something to touch** | The person needs a screen to enter their role, adjust their week, and read three results. | **Lovable** — three-screen web app: role entry → re-weight → results |
